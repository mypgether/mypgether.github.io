发表于 2016-07-30   |   分类于 Java   |  

jvm参数

  1. -Xmx Java Heap最大值，默认值为物理内存的1/4；
  2. -Xms Java Heap初始值，Server端JVM最好将-Xms和-Xmx设为相同值，开发测试机JVM可以保留默认值；
  3. -Xmn Java Heap Young区大小，不熟悉最好保留默认值；
  4. -Xss 每个线程的Stack大小，不熟悉最好保留默认值；
  5. -XX:PermSize：设定内存的永久保存区域；
  6. -XX:MaxPermSize：设定最大内存的永久保存区域；
  7. -XX:NewSize：设置JVM堆的‘新生代’的默认大小；
  8. -XX:MaxNewSize：设置JVM堆的‘新生代’的最大大小；

jps使用

jps 查看运行的java进程
jps -mlv

jstatd

    nohup jstatd -J-Djava.rmi.server.hostname=10.175.132.76 -J-Djava.security.policy=jstatd.all.policy -J-Dcom.sun.management.jmxremote.authenticate=false -J-Dcom.sun.management.jmxremote.ssl=false -J-Dcom.sun.management.jmxremote.port=8888 &
    启动一个守护线程
* java.rmi.server.hostname  jstatd绑定的主机ip地址
* com.sun.management.jmxremote.port  jmx的端口地址
* java.security.policy    赋予jstatd权限的文件

        jstatd.all.policy文件内容
        grant codebase "file:${java.home}/../lib/tools.jar" {
            permission java.security.AllPermission;
        }; 

* 实际如果需要通过VisualVM监控Tomcat，配置catalina.sh

        CATALINA_OPTS="-Djava.library.path=/usr/local/apr/lib -Dapplication.home=/usr/java/jdk1.8.0_91 -Djava.rmi.server.hostname=114.215.133.86 -Djava.security.policy=jstatd.all.policy -Dcom.sun.management.jmxremote.authenticate=false -Dcom.sun.management.jmxremote.ssl=false -Dcom.sun.management.jmxremote.port=8888"

javap

javap -l -p -c ConfigServiceImpl
1.l – 打印行数和局部变量
2.p – 打印包括非public在内的所有类和成员信息
3.c – 打印方法字节码(可以不添加)

jmap

点我查看

不需要引发 OutOfMemoryErrors 就可以获取堆dump文件的方法  
1. jmap -dump:live,format=b,file=heap.bin 1354  (sudo su)
2. 如果切换了用户出现这样的情况：
    VMVersionMismatchException: Supported versions are 23.25-b01. Target VM is 24.65-b04
    需要执行update-alternatives --get-selections | grep jinfo
        jinfo auto     /usr/lib/jvm/java-6-openjdk-amd64/bin/jinfo
    进行解决 [看我得到详细解答](http://serverfault.duapp.com/questions/653755/jinfo-for-java-7-openjdk-amd64) 
3. jhat heap.bin
4. http://localhost:7000

jhat

分析hprof文件工具
1. jvm配置oom的dump文件
    -XX:+HeapDumpOnOutOfMemoryError  -XX:HeapDumpPath=${目录} 
2. jhat java_pid5644.hprof  or  jhat heap.bin
3. http://localhost:7000

jstack

[Detail](http://www.hollischuang.com/archives/110)

实时生成hprof文件 
jstack -F -l 9153 然后通过jhat查看

jstat

实时监控heap size和垃圾回收状况  [1]()
        jstat -gcutil 17010 250 50   (17010为jvm pid，250表示250毫秒查看一次，50表示持续50次)
